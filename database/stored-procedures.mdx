## Login module

`02_procedimientos_Login.sql`

<AccordionGroup>

  <Accordion title="sp_login_usuario — Authenticate a user">

  Validates a user's credentials and returns their profile on success, or a descriptive error message on failure.

  | Parameter | Direction | Type | Description |
  |-----------|-----------|------|-------------|
  | `p_matricula` | IN | `VARCHAR(50)` | User's enrollment ID |
  | `p_password` | IN | `VARCHAR(255)` | Plain-text password (hashed internally) |

  **Returns on success:** `success=1`, `message`, `id_usuario`, `matricula`, `id_rol`, `id_turno`, `nombre`, `apellido_paterno`, `apellido_materno`

  **Returns on failure:** `success=0`, `message` (`'Usuario no encontrado o inactivo'` or `'Contraseña incorrecta'`)

  ```sql
  CREATE PROCEDURE sp_login_usuario(
      IN p_matricula VARCHAR(50),
      IN p_password VARCHAR(255)
  )
  BEGIN
      DECLARE v_password_hash VARCHAR(64);
      DECLARE v_id_usuario INT;

      -- Buscar usuario activo
      SELECT id_usuario, password_hash
      INTO v_id_usuario, v_password_hash
      FROM tbl_usuarios
      WHERE matricula = p_matricula
        AND id_estatus = 'Activo'
      LIMIT 1;

      -- Si no existe usuario activo
      IF v_id_usuario IS NULL THEN
          SELECT 
              0 AS success,
              'Usuario no encontrado o inactivo' AS message;
      ELSE
          -- Comparar hash
          IF v_password_hash = SHA2(p_password, 256) THEN

              SELECT 
                  1 AS success,
                  'Login correcto' AS message,
                  u.id_usuario,
                  u.matricula,
                  u.id_rol,
                  u.id_turno,
                  p.nombre,
                  p.apellido_paterno,
                  p.apellido_materno
              FROM tbl_usuarios u
              INNER JOIN tbl_personas p 
                  ON p.id_persona = u.id_persona
              WHERE u.id_usuario = v_id_usuario;

          ELSE
              SELECT 
                  0 AS success,
                  'Contraseña incorrecta' AS message;
          END IF;
      END IF;

  END
  ```

  <Note>
  Only users with `id_estatus = 'Activo'` can log in. Inactive or soft-deleted users are rejected before the password check.
  </Note>

  </Accordion>

</AccordionGroup>

## Salones module

`05_CRUD_ModuloSalones.sql`

<AccordionGroup>

  <Accordion title="sp_insertar_salon — Create a new room">

  Inserts a new room inside a building. Uses a transaction and validates that the building exists and that the room name is not already taken. On success it also increments `tbl_edificios.total_salones`.

  | Parameter | Direction | Type | Description |
  |-----------|-----------|------|-------------|
  | `p_id_edificio` | IN | `INT` | Target building ID |
  | `p_tipo_salon` | IN | `VARCHAR(20)` | `'Salon'`, `'Laboratorio'`, or `'Sala'` |
  | `p_nombre` | IN | `VARCHAR(50)` | Unique room name |
  | `p_capacidad_personas` | IN | `INT` | Seating capacity |
  | `p_estatus` | IN | `VARCHAR(20)` | Initial status (`'Activo'`, etc.) |

  **Returns:** `mensaje` (description), `id_salon` (0 on error, new ID on success)

  ```sql
  CREATE PROCEDURE sp_insertar_salon(
      IN p_id_edificio INT,
      IN p_tipo_salon VARCHAR(20),
      IN p_nombre VARCHAR(50),
      IN p_capacidad_personas INT,
      IN p_estatus VARCHAR(20)
  )
  BEGIN
      DECLARE v_id_salon INT;
      DECLARE v_edificio_existe INT;
      DECLARE v_nombre_existe INT;
      
      DECLARE EXIT HANDLER FOR SQLEXCEPTION
      BEGIN
          ROLLBACK;
          SELECT 'Error al insertar el salón' AS mensaje, 0 AS id_salon;
      END;
      
      START TRANSACTION;
      
      SELECT COUNT(*) INTO v_edificio_existe 
      FROM tbl_edificios 
      WHERE id_edificio = p_id_edificio AND id_estatus = 'Activo';
      
      IF v_edificio_existe = 0 THEN
          SELECT 'Error: El edificio no existe o no está activo' AS mensaje, 0 AS id_salon;
          ROLLBACK;
      ELSE
          SELECT COUNT(*) INTO v_nombre_existe 
          FROM tbl_salones 
          WHERE nombre = p_nombre;
          
          IF v_nombre_existe > 0 THEN
              SELECT 'Error: Ya existe un salón con ese nombre' AS mensaje, 0 AS id_salon;
              ROLLBACK;
          ELSE
              INSERT INTO tbl_salones (
                  id_edificio,
                  id_tipo_salon,
                  nombre,
                  capacidad_personas,
                  id_estatus
              ) VALUES (
                  p_id_edificio,
                  p_tipo_salon,
                  p_nombre,
                  p_capacidad_personas,
                  p_estatus
              );
              
              SET v_id_salon = LAST_INSERT_ID();
              
              UPDATE tbl_edificios 
              SET total_salones = total_salones + 1 
              WHERE id_edificio = p_id_edificio;
              
              COMMIT;
              
              SELECT 'Salón agregado correctamente' AS mensaje, v_id_salon AS id_salon;
          END IF;
      END IF;
  END
  ```

  </Accordion>

  <Accordion title="sp_actualizar_salon — Update a room">

  Updates all editable fields of a room record. Validates that the room exists, the target building is active, and that the new name does not conflict with another room.

  | Parameter | Direction | Type | Description |
  |-----------|-----------|------|-------------|
  | `p_id_salon` | IN | `INT` | Room to update |
  | `p_id_edificio` | IN | `INT` | New building ID |
  | `p_tipo_salon` | IN | `VARCHAR(20)` | New type |
  | `p_nombre` | IN | `VARCHAR(50)` | New unique name |
  | `p_capacidad_personas` | IN | `INT` | New capacity |
  | `p_estatus` | IN | `VARCHAR(20)` | New status |

  **Returns:** `mensaje`, `id_salon`

  ```sql
  CREATE PROCEDURE sp_actualizar_salon(
      IN p_id_salon INT,
      IN p_id_edificio INT,
      IN p_tipo_salon VARCHAR(20),
      IN p_nombre VARCHAR(50),
      IN p_capacidad_personas INT,
      IN p_estatus VARCHAR(20)
  )
  BEGIN
      DECLARE v_edificio_existe INT;
      DECLARE v_nombre_existe INT;
      DECLARE v_salon_existe INT;
      
      DECLARE EXIT HANDLER FOR SQLEXCEPTION
      BEGIN
          ROLLBACK;
          SELECT 'Error al actualizar el salón' AS mensaje, 0 AS id_salon;
      END;
      
      START TRANSACTION;
      
      SELECT COUNT(*) INTO v_salon_existe 
      FROM tbl_salones 
      WHERE id_salon = p_id_salon;
      
      IF v_salon_existe = 0 THEN
          SELECT 'Error: El salón no existe' AS mensaje, 0 AS id_salon;
          ROLLBACK;
      ELSE
          SELECT COUNT(*) INTO v_edificio_existe 
          FROM tbl_edificios 
          WHERE id_edificio = p_id_edificio AND id_estatus = 'Activo';
          
          IF v_edificio_existe = 0 THEN
              SELECT 'Error: El edificio no existe o no está activo' AS mensaje, 0 AS id_salon;
              ROLLBACK;
          ELSE
              SELECT COUNT(*) INTO v_nombre_existe 
              FROM tbl_salones 
              WHERE nombre = p_nombre AND id_salon != p_id_salon;
              
              IF v_nombre_existe > 0 THEN
                  SELECT 'Error: Ya existe otro salón con ese nombre' AS mensaje, 0 AS id_salon;
                  ROLLBACK;
              ELSE
                  UPDATE tbl_salones 
                  SET 
                      id_edificio = p_id_edificio,
                      id_tipo_salon = p_tipo_salon,
                      nombre = p_nombre,
                      capacidad_personas = p_capacidad_personas,
                      id_estatus = p_estatus
                  WHERE id_salon = p_id_salon;
                  
                  COMMIT;
                  
                  SELECT 'Salón actualizado correctamente' AS mensaje, p_id_salon AS id_salon;
              END IF;
          END IF;
      END IF;
  END
  ```

  </Accordion>

  <Accordion title="sp_eliminar_salon — Soft-delete a room">

  Sets `id_estatus = 'Eliminado'` on the specified room. Rejects the operation if the room does not exist or is already deleted.

  | Parameter | Direction | Type | Description |
  |-----------|-----------|------|-------------|
  | `p_id_salon` | IN | `INT` | Room to delete |

  **Returns:** `mensaje`

  ```sql
  CREATE PROCEDURE sp_eliminar_salon(
      IN p_id_salon INT
  )
  BEGIN
      DECLARE v_salon_existe INT;
      DECLARE v_estatus_actual VARCHAR(20);
      
      DECLARE EXIT HANDLER FOR SQLEXCEPTION
      BEGIN
          ROLLBACK;
          SELECT 'Error al eliminar el salón' AS mensaje;
      END;
      
      START TRANSACTION;
      
      SELECT COUNT(*), id_estatus INTO v_salon_existe, v_estatus_actual
      FROM tbl_salones 
      WHERE id_salon = p_id_salon
      GROUP BY id_estatus;
      
      IF v_salon_existe = 0 THEN
          SELECT 'Error: El salón no existe' AS mensaje;
          ROLLBACK;
      ELSE
          IF v_estatus_actual = 'Eliminado' THEN
              SELECT 'El salón ya se encuentra eliminado' AS mensaje;
              ROLLBACK;
          ELSE
              UPDATE tbl_salones 
              SET id_estatus = 'Eliminado'
              WHERE id_salon = p_id_salon;
              
              COMMIT;
              
              SELECT 'Salón eliminado correctamente' AS mensaje;
          END IF;
      END IF;
  END
  ```

  </Accordion>

</AccordionGroup>

  <Accordion title="sp_registrar_inventario — Register a new inventory item">

  Creates a new furniture/equipment record. Validates that the `id_mobiliario` code is unique and that the specified room belongs to the specified building before inserting.

  | Parameter | Direction | Type | Description |
  |-----------|-----------|------|-------------|
  | `p_id_salon` | IN | `INT` | Room assignment |
  | `p_id_mobiliario` | IN | `VARCHAR(10)` | Unique asset code (e.g. `MOB001`) |
  | `p_id_edificio` | IN | `INT` | Building assignment |
  | `p_id_ubicacion` | IN | `ENUM('Edificio','Administracion')` | Physical location type |
  | `p_id_tipo_mobi` | IN | `ENUM(...)` | Furniture type |
  | `p_id_categoria_mobi` | IN | `ENUM(...)` | Category |
  | `p_id_estatus` | IN | `ENUM('Activo','Inactivo','Eliminado')` | Lifecycle status |
  | `p_id_estatus_actual` | IN | `ENUM('Disponible','En uso')` | Operational state |
  | `p_observaciones` | IN | `TEXT` | Free-text notes |

  **Returns:** `mensaje`

  ```sql
  CREATE PROCEDURE sp_registrar_inventario(
      IN p_id_salon INT,
      IN p_id_mobiliario VARCHAR(10),
      IN p_id_edificio INT,
      IN p_id_ubicacion ENUM('Edificio','Administracion'),
      IN p_id_tipo_mobi ENUM('Silla','Mesa','Pizarron','Ventana','Cañon','Pantalla','Extension','HDMI','Proyector'),
      IN p_id_categoria_mobi ENUM('Tecnología','Cables','Eléctrico','Papelería'),
      IN p_id_estatus ENUM('Activo','Inactivo','Eliminado'),
      IN p_id_estatus_actual ENUM('Disponible','En uso'),
      IN p_observaciones TEXT
  )
  BEGIN
      DECLARE v_existe_relacion INT;
      DECLARE v_existe_mobiliario INT;

      SELECT COUNT(*) INTO v_existe_mobiliario 
      FROM tbl_inventario 
      WHERE id_mobiliario = p_id_mobiliario;

      IF v_existe_mobiliario > 0 THEN
          SIGNAL SQLSTATE '45000' 
          SET MESSAGE_TEXT = 'Error: El ID de mobiliario ya existe en el sistema.';
      ELSE
          SELECT COUNT(*) INTO v_existe_relacion 
          FROM tbl_salones 
          WHERE id_salon = p_id_salon AND id_edificio = p_id_edificio;

           IF v_existe_relacion > 0 THEN
              INSERT INTO tbl_inventario (
                  id_salon, id_mobiliario, id_edificio, id_ubicacion,
                  id_tipo_mobi, id_categoria_mobi, id_estatus,
                  id_estatus_actual, observaciones
              ) 
              VALUES (
                  p_id_salon, p_id_mobiliario, p_id_edificio, p_id_ubicacion,
                  p_id_tipo_mobi, p_id_categoria_mobi, p_id_estatus,
                  p_id_estatus_actual, p_observaciones
              );
              
              SELECT 'Registro insertado con éxito' AS mensaje;
          ELSE
              SIGNAL SQLSTATE '45000' 
              SET MESSAGE_TEXT = 'Error: El salón no pertenece al edificio seleccionado.';
           END IF;
      END IF;
  END
  ```

  </Accordion>

  Returns all active inventory items of a given type, joined with their room and building information.

  |-----------|-----------|------|-------------|

  ```sql
  CREATE PROCEDURE sp_getDetalleInventarioPorTipo(
      IN p_tipo_mobi VARCHAR(50) 
  )
  BEGIN
      SELECT
          i.id_inventario,
          i.id_mobiliario,
          i.id_tipo_mobi,

      WHERE i.id_estatus = 'Activo'

  END
  ```

## Préstamos module

`08_Modulo_Prestamos.sql`

<AccordionGroup>

  <Accordion title="sp_insertar_prestamo — Create an equipment loan">

  Resolves the user by their `matricula` and the inventory item by its `id_mobiliario` code, then inserts the loan record with status `'Activo'`. Raises a SQLSTATE `45000` error if either lookup fails.

  After the insert, the trigger `tg_actualizar_inventario_salida` automatically sets `id_estatus_actual = 'En Uso'` on the borrowed item.

  | Parameter | Direction | Type | Description |
  |-----------|-----------|------|-------------|
  | `p_matricula` | IN | `VARCHAR(25)` | Borrower's enrollment ID |
  | `p_id_mobiliario_text` | IN | `VARCHAR(25)` | Asset code of the item being lent |
  | `p_fecha_prevista` | IN | `DATETIME` | Expected return date/time |
  | `p_obs_salida` | IN | `TEXT` | Condition notes at check-out |

  ```sql
  CREATE PROCEDURE sp_insertar_prestamo(
      IN p_matricula VARCHAR(25),
      IN p_id_mobiliario_text VARCHAR(25),
      IN p_fecha_prevista DATETIME,
      IN p_obs_salida TEXT
  )
  BEGIN
      DECLARE v_id_usuario INT;
      DECLARE v_id_inv INT;

      SELECT id_usuario INTO v_id_usuario 
      FROM tbl_usuarios 
      WHERE matricula = p_matricula LIMIT 1;

      SELECT id_inventario INTO v_id_inv 
      FROM tbl_inventario 
      WHERE id_mobiliario = p_id_mobiliario_text LIMIT 1;

      IF v_id_usuario IS NOT NULL AND v_id_inv IS NOT NULL THEN
          INSERT INTO tbl_prestamos_mobiliario (
              id_usuario, 
              id_inventario, 
              fecha_devolucion_prevista, 
              observaciones_salida, 
              estatus_prestamo
          )
          VALUES (v_id_usuario, v_id_inv, p_fecha_prevista, p_obs_salida, 'Activo');
      ELSE
          SIGNAL SQLSTATE '45000'
          SET MESSAGE_TEXT = 'Error: Usuario o Mobiliario no encontrado';
      END IF;
  END
  ```

  </Accordion>

  <Accordion title="sp_actualizar_estatus_prestamo — Update a loan's status">

  Changes the status of a loan (e.g. to `'Devuelto'`, `'Atrasado'`, `'Extraviado'`). When the new status is `'Devuelto'`, `fecha_devolucion_real` is automatically set to the current timestamp.

  | Parameter | Direction | Type | Description |
  |-----------|-----------|------|-------------|
  | `p_id_prestamo` | IN | `INT` | Loan record ID |
  | `p_nuevo_estatus` | IN | `ENUM('Activo','Devuelto','Atrasado','Extraviado','Eliminado')` | New status |
  | `p_observaciones_entrada` | IN | `TEXT` | Condition notes at return |

  **Returns:** `mensaje`, `id`

  ```sql
  CREATE PROCEDURE sp_actualizar_estatus_prestamo(
      IN p_id_prestamo INT,
      IN p_nuevo_estatus ENUM('Activo', 'Devuelto', 'Atrasado', 'Extraviado', 'Eliminado'),
      IN p_observaciones_entrada TEXT
  )
  BEGIN
      DECLARE v_fecha_devolucion DATETIME DEFAULT NULL;

      IF p_nuevo_estatus = 'Devuelto' THEN
          SET v_fecha_devolucion = NOW();
      END IF;

      UPDATE tbl_prestamos_mobiliario
      SET 
          estatus_prestamo       = p_nuevo_estatus,
          observaciones_entrada  = p_observaciones_entrada,
          fecha_devolucion_real  = IF(p_nuevo_estatus = 'Devuelto', v_fecha_devolucion, fecha_devolucion_real)
      WHERE id_prestamo = p_id_prestamo;

      SELECT 'Préstamo actualizado correctamente' AS mensaje, p_id_prestamo AS id;
  END
  ```

  </Accordion>

  <Accordion title="sp_eliminar_Prestamo — Soft-delete a loan">

  Marks a loan as `'Eliminado'`.

  | Parameter | Direction | Type | Description |
  |-----------|-----------|------|-------------|
  | `p_id_prestamo` | IN | `INT` | Loan record ID |


## Triggers

Three triggers keep `tbl_bitacora` and `tbl_inventario` synchronized automatically without requiring any application-layer logic.

<AccordionGroup>

  <Accordion title="trg_nueva_reservacion — Log new reservations">

  Fires **AFTER INSERT** on `tbl_reservaciones`. Looks up the room name and the author's full name, then appends a row to `tbl_bitacora`.

  ```sql
  CREATE TRIGGER trg_nueva_reservacion
  AFTER INSERT ON tbl_reservaciones
  FOR EACH ROW
  BEGIN
      DECLARE v_nombre_salon VARCHAR(50);
      DECLARE v_autor VARCHAR(100);
      
      SELECT nombre INTO v_nombre_salon
      FROM tbl_salones
      WHERE id_salon = NEW.id_salon;
      
      SELECT CONCAT(u.matricula, ' - ', p.nombre, ' ', p.apellido_paterno) INTO v_autor
      FROM tbl_usuarios u
      INNER JOIN tbl_personas p ON u.id_persona = p.id_persona
      WHERE u.id_usuario = NEW.id_usuario;
      
      INSERT INTO tbl_bitacora (titulo, descripcion, autor)
      VALUES (
          'Nueva reservación',
          CONCAT('Se reservó el salón: ', v_nombre_salon, ' para la fecha: ', NEW.fecha_hora),
          v_autor
      );
  END
  ```

  **Bitácora entry format:**
  - `titulo`: `'Nueva reservación'`
  - `descripcion`: `'Se reservó el salón: <nombre> para la fecha: <fecha_hora>'`
  - `autor`: `'<matricula> - <nombre> <apellido_paterno>'`

  </Accordion>

  <Accordion title="trg_nuevo_reporte — Log new damage reports">

  Fires **AFTER INSERT** on `tbl_reportes_mobiliario`. Looks up the author's identity and appends a row to `tbl_bitacora` with the report type.

  ```sql
  CREATE TRIGGER trg_nuevo_reporte
  AFTER INSERT ON tbl_reportes_mobiliario
  FOR EACH ROW
  BEGIN
      DECLARE v_autor VARCHAR(100);
      
      SELECT CONCAT(u.matricula, ' - ', p.nombre, ' ', p.apellido_paterno) INTO v_autor
      FROM tbl_usuarios u
      INNER JOIN tbl_personas p ON u.id_persona = p.id_persona
      WHERE u.id_usuario = NEW.id_usuario;
      
      INSERT INTO tbl_bitacora (titulo, descripcion, autor)
      VALUES (
          'Nuevo reporte creado',
          CONCAT('Tipo: ', NEW.id_tipo_reporte),
          v_autor
      );
  END
  ```

  **Bitácora entry format:**
  - `titulo`: `'Nuevo reporte creado'`
  - `descripcion`: `'Tipo: <id_tipo_reporte>'`
  - `autor`: `'<matricula> - <nombre> <apellido_paterno>'`

  </Accordion>

  <Accordion title="tg_actualizar_inventario_salida — Mark item as in use on loan">

  Fires **AFTER INSERT** on `tbl_prestamos_mobiliario`. Immediately updates the borrowed item's `id_estatus_actual` to `'En Uso'` in `tbl_inventario`.

  ```sql
  CREATE TRIGGER tg_actualizar_inventario_salida
  AFTER INSERT ON tbl_prestamos_mobiliario
  FOR EACH ROW
  BEGIN
      UPDATE tbl_inventario
      SET id_estatus_actual = 'En Uso'
      WHERE id_inventario = NEW.id_inventario;
  END
  ```

  <Info>
  The reverse transition — setting `id_estatus_actual` back to `'Disponible'` on return — is not handled by a trigger. It must be managed at the application level when `sp_actualizar_estatus_prestamo` is called with `p_nuevo_estatus = 'Devuelto'`.
  </Info>

  </Accordion>

</AccordionGroup>
