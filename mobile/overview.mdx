The GAM mobile app is built with **Ionic 8** and **Angular 20**, deployable to iOS and Android via **Capacitor 8**. It gives staff and administrators a cross-platform interface to manage furniture inventory, room reservations, loans, classroom data, and maintenance reports — all connected to the same Java back end as the web application.

## Technology stack

| Layer | Technology | Version |
|---|---|---|
| UI framework | Ionic Angular | ^8.0.0 |
| Application framework | Angular | ^20.0.0 |
| Native runtime | Capacitor | 8.1.0 |
| HTTP client | Angular HttpClient | — |
| Icons | Ionicons | ^7.0.0 |
| Reactive streams | RxJS | ~7.8.0 |
| Language | TypeScript | ~5.9.0 |

<Note>
  The app uses Angular's lazy-loaded module strategy (`PreloadAllModules`) so every feature module is bundled separately and preloaded in the background after the initial render.
</Note>

## Platform support

Capacitor wraps the compiled Angular/Ionic web app into a native shell. The same codebase targets:

- **Android** — via Android Studio and the Capacitor Android plugin
- **iOS** — via Xcode and the Capacitor iOS plugin
- **Browser** — full Progressive Web App support for development and desktop use

The Capacitor app ID is `io.ionic.starter` and the web output directory is `www`.

## Application structure

The entry point redirects immediately to the `home` route. All feature modules are lazy-loaded from `src/app/Modulos/`:

```
src/app/
├── app.module.ts            # Root module — IonicModule, HttpClientModule
├── app-routing.module.ts    # Top-level lazy routes
├── components/
│   ├── sidebar/             # Global navigation sidebar
│   └── header/              # Shared page header
├── Modelos/
│   ├── salones.model.ts     # Salones interface
│   └── usuario.ts           # LoginRequest / LoginResponse interfaces
└── Modulos/
    ├── dashboard/
    ├── inventarios/
    ├── inventario-reportes/
    ├── prestamos/
    ├── reservaciones/
    ├── reportes/
    └── salones/
```

## API connectivity

In development, all requests to `/api/*` are proxied to the Java back end by `proxy.conf.json`. The `environment.ts` file sets `apiUrl` to `'/api'` so services never hard-code the origin.

```json proxy.conf.json
{
  "/api": {
    "target": "http://localhost:8080/IntegradoraII_Web",
    "secure": false,
    "changeOrigin": true,
    "logLevel": "debug"
  }
}
```

## Native Android app (legacy)

The repository also contains a separate native Android application at `02_Movil/app/` built with the standard Android SDK (`AppCompatActivity`). It includes the following activities:

| Activity | Purpose |
|---|---|
| `Login` | Login screen |
| `MainActivity` | Main entry point |
| `DashboardActivity` | Dashboard view |
| `InventarioActivity` | Inventory list |
| `SalonesActivity` | Classroom list |
| `ReservacionesActivity` | Reservations view |
| `ReportesDaniosActivity` | Damage reports |
| `ReportesInventarioActivity` | Inventory reports |

<Note>
  This native Android app is an earlier prototype. The primary mobile client for GAM is the **Ionic/Angular app** documented in this section, which supports both iOS and Android through Capacitor.
</Note>
